До - класс проверки завершенности онбординга.
Есть три условных проверки - общая завершенность IsCompleted, проверка завершенности user flow IsUserFlowCompleted, проверка завершенности конкретной точки.
Взял класс, который не успел привести в нормальный вид, из-за загрузки.


После - на самом деле есть 3 точки для открытия счета
- Завершенность юзер флоу (завершены все важные шаги) - обязательно
- Завершенность прайсинга (прайсинг выбран из бэкофиса сейлсом для корпоратов или юзером на экране в онбординге) - обязательно
- Завершенность подписания (опционально для некоторых онбордингов) - необязательно

Использованный прием - полиморфизм + изменение подхода - сделал явными опциональные точки.
Переместил каждую условную проверку некоторой неизвестной величиной туда, где она уже будет известна.
Ну и также изменил ментальную модель, чтобы коду стали не нужны условные проверки.
