1) DuplicatedCode
Тяжело было читать большой метод + не понравились if и switch одновременно - 
добавил явно необходимый полиморфизм через функции заполнения, оставив switch (не хотел в этот раз создавать классов внутри). 

2) Classes with too much code
Моделька в которой намешаны все возможные состояния, и не понятно какое реально может существовать отдельно.
Добавил типизацию, чтобы понимать с какой "засабмиченной" моделью реально работаем на данный момент.

3) Comparing variables to null
Избавляемся от null через Option NullableObject.

4) Classes with too many instance variables
Есть явная дата шедулинга, со своими ограничениями и временем экспирации.
Вынес ее в условный value object, чтобы не иметь возможности создавать не валидную дату. 

5) Code not actually ever used

В базовом классе работы с шагом есть ShowCloseButton и другие методы, которые могут быть просто не использованы.
Перемещаем это дело в отдельный интерфейс, и наследуемся при необходимости.

Итого - на данном занятии убедился, насколько важны постоянная практика и обучение, "запашки" заметил быстро и
мозг сразу выдавал несколько вариантов решения.
При этом вспоминаю, как раньше я был неуверен в свои интуитивных решениях, 
поскольку не было должной подготовки.
