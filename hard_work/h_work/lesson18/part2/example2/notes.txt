Есть тип (Машина из Crm) и при создании временного пропуска, есть интересная валидация,
которая пропускает Машины с определенным id, в спецификации этого конечно не указано.
оказывается, для таких машин, просто нет записей в CRM.

При этом логика проверки не совпадения номера машины из CRM и номера машины из сущности - остаются.

Явно указываем в интерфейсе, какие могут быть причины не создания временного пропуска в результате 
+ передаем машину из CRM и сам запрос напрямую в интерфейс - явно.
